#  Práctica 2 – Gestión Colaborativa de Material Académico con Git y GitHub

## UT4 – Optimización y Documentación

Repositorio correspondiente a la **Práctica 2 de la Unidad de Trabajo 4**, cuyo objetivo es organizar, documentar y gestionar de forma colaborativa el material académico utilizando **Git y GitHub**.

Este repositorio centraliza todos los recursos generados durante el desarrollo de las asignaturas **Lenguaje de Marcas** y **Entornos de Desarrollo**, permitiendo mantener una estructura clara, control de versiones y acceso organizado al contenido.

El proyecto consiste en la creación de un repositorio estructurado que almacene y documente el material de estudio de ambas asignaturas.

* Apuntes.

#  Archivo .gitignore

El repositorio incluye un archivo **.gitignore** que permite indicar a Git qué archivos o carpetas no deben ser rastreados ni subidos al repositorio.

Este archivo es importante para evitar subir:

* Archivos temporales del sistema
* Dependencias innecesarias (por ejemplo `node_modules/`)
* Archivos generados automáticamente
* Archivos de configuración local
* Credenciales o claves API
* Archivos de compilación

El uso correcto del `.gitignore` permite mantener el repositorio **limpio, seguro y optimizado**.
